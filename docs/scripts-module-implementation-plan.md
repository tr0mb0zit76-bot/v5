# План внедрения модуля «Скрипты» (продажи, экспедиция)

Документ задаёт **последовательность работ** и **критерии готовности** этапов. Отклонения от плана фиксируем отдельно (изменение scope).

**Стек:** Laravel 13, Inertia v2, Vue 3, PHPUnit, существующие роли/видимость — переиспользовать.

---

## Принципы

1. Сначала **данные и сессии** (что логируем), затем **аналитика**, затем **«умные» подсказки**.
2. Обучение на старте = **агрегаты по событиям** (вероятности, частоты), без ML до накопления объёма.
3. Скрипт = **граф узлов + переходы по классам реакций клиента**, не просто текст.
4. Каждый этап заканчивается **тестами** и **демо сценарием** для отдела продаж.

## Фаза 0. Подготовка (1 короткий спринт)

| # | Задача | Готово, когда |
|---|--------|----------------|
| 0.1 | Утвердить **1–2 пилотных сценария** (например: «первичный запрос ставки», «возражение: дорого») с отделом продаж | Есть тексты и ветки на бумаге/Miro |
| 0.2 | Описать **enum исходов** сессии (`no_contact`, `progress`, `quote_sent`, `won`, `lost`, `postponed` — скорректировать под вашу воронку) | Список в коде/миграции |
| 0.3 | Описать **классы реакций клиента** (10–20 штук на старте): `price_objection`, `need_time`, `competitor`, `clarify_route`, … | Справочник в БД или config + сиды |
| 0.4 | Решить **где открывать модуль** (страница «Скрипты», виджет в карточке контрагента, ссылка из заказа) | Согласованный UX-скетч |

**Артефакт:** короткое ТЗ на пилот (1 страница).

## Фаза 5. Аналитика и «обучение» (статистика)

| # | Задача | Готово, когда |
|---|--------|----------------|
| 5.1 | Агрегаты: по `(script_version, node_id, reaction_class)` — количество переходов, доля исходов `won` / `progress` (настроить окно: например 30 дней) | SQL/Query + сервис |

## Тестирование (на всех фазах)

- Feature-тесты: создание сценария, проход, завершение, запрет доступа чужой роли.
- Минимум один **сквозной тест**: сид → открыть версию → 3 перехода → исход → в БД ожидаемые строки в `script_play_events`.
- После изменений PHP: `vendor/bin/pint --dirty`.

## Definition of Done (весь модуль «пилот готов»)

- Менеджер проходит **2 утверждённых сценария** и фиксирует исход.
- Руководитель видит **отчёт по возражениям и узлам**.
- В прохождении отображается **статистическая подсказка** там, где данных достаточно.
- Документированы роли и место модуля в навигации.
